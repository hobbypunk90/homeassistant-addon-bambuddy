# Bambuddy – Home Assistant App Repository

[Bambuddy](https://github.com/maziggy/bambuddy) wrapped inside a Home Assistant App (formerly Add-on), with companion Slicer API and Failure Detection sidecars.

A self-hosted print archive and management system for Bambu Lab 3D printers, with NEW automatic slicing compatibility. For easy deployment on HAOS devices.

## What This App Repository Provides

This repository delivers a **multi-app ecosystem** for running Bambuddy on Home Assistant OS with minimal setup:

### 🧩 Four Installable Apps
| App | Default Port | Purpose |
|--------|------|---------|
| **Bambuddy** | 8000 | Core application — printer monitoring, archive, queue, notifications |
| **Slicer API (Bambu Studio)** | 3001 | Optional headless slicing sidecar running Bambu Studio CLI |
| **Slicer API (OrcaSlicer)** | 3003 | Optional headless slicing sidecar running OrcaSlicer CLI |
| **Obico ML API** | 3333 | Optional headless ML sidecar for AI failure detection |

Install the main add-on **and** any optional sidecars you need.
(Also recommended up front is the incredibly fully featured [HA Bambu Lab Integration](greghesp/ha-bambulab) for printer monitoring and control directly within Home Assistant; several additional entities are derived below to complement the printer controls and sensors)

### 🔌 Zero-Config Setup
- **Home Assistant API**: API connectivity is automatic. The Supervisor injects the necessary tokens at startup. Just configure your notifications in Bambuddy's `Settings` > `Notifications` and you're good to go!
- **MQTT**: If the [Mosquitto add-on](https://github.com/home-assistant/addons/tree/master/mosquitto) is installed, broker credentials are auto-detected from the Supervisor. It just works—no manual configuration required.
- **Timezone**: Automatically injected by the Supervisor.

### ⚙️ Local "One-Click" Slicing from file: Orca and/or Bambu CLI Endpoints
- **Automatic slicing in browser** – Bambuddy's recent addition of local headless slicing on-demand as of v0.2.4 inspired the compilation of these apps for HAOS installation.
- Choice between **Orca Slicer and Bambu Studio** – Install either or both sidecar containers to choose your preferred automatic slicer and continue to replace Bambu Handy functionality and automate slicing using your presets. Default slicer URLs point to `http://127.0.0.1:3001` / `:3003` for the co-located sidecars. Override in the add-on config to point at remote slicer instances if needed.
- Easily choose your preferred slicer within Bambuddy's UI via `Settings` > `Workflow`.
- A Gear 
- **Note current bug/limitation of latest versions of Orca Slicer** which may be unable to properly interpret and slice .3mf files generated by Bambu Studio; See Bambuddy documentation for recommendations (selecting Bambu Studio may be currently recommended in several common scenarios).


### 🖼️ Empty Plate Detection (OpenCV)
The Docker image was modified to use a **Python 3.11 base** specifically tailored to match Home Assistant's precompiled [`opencv-python-headless`](https://wheels.home-assistant.io/musllinux/) wheels. This means:
- **~5 minute install** on adequately powered machines, instead of a 30+ minute source compilation that usually fails
- Empty plate detection works out of the box (though it is inherently finicky. Set this up on the Printer card on the Bambuddy web UI.

### 🍝 AI Failure Detection (Obico ML)
- ** Spaghetti Detection ** – Bambuddy can monitor your print's camera feed in real-time and detect failures (spaghetti, layer shifts, etc.) via a self-hosted Obico ML server (Formerly Spaghetti Detective). To enable this, simply install the **Obico ML API** add-on provided in this repository alongside Bambuddy. *(Note: The Obico ML app is fundamentally based on the excellent work by [nobodyguy](https://github.com/nobodyguy/obico_ml_ha_addon)), and is provided in this repository for convenience.*
- **Note:** This is useful for **all** Bambu printers. While some models (X1C, P1S) have built-in LIDAR for first-layer inspection, continuous AI print monitoring is a Bambu Cloud feature that is **not available in developer/LAN mode**. Obico ML fills that gap locally.
- The included **Obico ML API** add-on exposes a customized endpoint (`/detect/`) that automatically draws visual bounding boxes around detected failures, passing them directly into Bambuddy's Home Assistant push notifications so you can easily see what went wrong. This can be enabled/disabled via the HAOS App Config tab.

- **Smart Failure Monitoring Sensors** for easy automations – most useful as a complement to the incredibly fully featured [HA Bambu Lab Integration](greghesp/ha-bambulab) which is recommended for printer monitoring and control directly within Home Assistant
- Enhanced Bambuddy by injecting three MQTT binary sensors set to **auto-discovery in Home Assistant** based on Bambuddy's internal Obico-based Failure Detection. [experimental] They should automatically appear in your MQTT integration in HA, allowing you to build clean automations (like controlling chamber lights) based on the start or end of failure detection or the detection of a failure, allowing for flexible control during print pauses automated by Bambuddy upon failure (in addition to automations based on printing status as already offered by [HA Bambu Lab Integration](greghesp/ha-bambulab).
- `binary_sensor.bambuddy_<serial>_obico_active`: ON ONLY while actively checking photos (turns off immediately when paused).
- `binary_sensor.bambuddy_<serial>_obico_failed`: ON ONLY if a spaghetti failure is currently latched (stays true during the pause, clears on resume).
- `binary_sensor.bambuddy_<serial>_obico_monitoring`: ON if Obico is active OR a failure is latched.
- Use the unified `obico_monitoring` sensor as a single condition (`state == 'on'`) to seamlessly keep your lights on while the printer runs, and *stay on* while investigating a failure. Use `obico_active` when lights off upon an error detection is preferred. Use 


## Limitations inherent to Bambuddy:

### ⚠️ Host Network Mode Required – Check port assignments
Bambuddy runs in `host_network: true` mode. This is **required** (by hard-coded expectations in Bambu Lab printers Bambu Studio, Orca Slicer, etc.) for:
- SSDP printer discovery
- Virtual Printer FTP file transfers
- Camera streaming

**Port conflict warning** — Bambuddy binds several ports on the host which cannot be easily reconfigured (8000, 8883, 3000, 3002, 322, 990, 2021, 6000; temporarily will use a given port in the range 50000–50100 during file transfers i.e. upon print initiation). Review the [DOCS.md](./bambuddy/DOCS.md) for the full list and check for conflicts with other add-ons (notably will need to change from default port assignments for core add-ons including Z-Wave JS/JS UI on port 3000, and Mosquitto TLS on port 8883).

### 🚫 Ingress Not Supported
Bambuddy is accessed directly via its web UI at `http://<your-ha-ip>:8000` — it **cannot** run behind Home Assistant's Ingress iframe proxy. The frontend uses fixed root-context paths and WebSocket connections that are incompatible with Ingress's dynamic `/api/hassio_ingress/<token>/` URL rewriting. This would require significant upstream changes to Bambuddy's routing architecture. [Bambuddy](maziggy/Bambuddy) is natively compatible with Tailscale run on the host (e.g., via the official [Tailscale HA Community App](hassio-addons/app-tailscale)) to provide a straightforward path to remote access.

## Installation
[![Open your Home Assistant instance and show the add app repository dialog with a specific repository URL pre-filled.](https://my.home-assistant.io/badges/supervisor_add_addon_repository.svg)](https://my.home-assistant.io/redirect/supervisor_add_addon_repository/?repository_url=https%3A%2F%2Fgithub.com%2Fconnormxy%2Fhomeassistant-addon-bambuddy#main)

1. Add this repository URL to your Home Assistant add-on store. Click the above link, or:
In Home Assistant, open `⚙️ Settings` > `🧩 Apps` > `🏠+ Install App` > `… Menu` > `Repositories` > `➕ Add` and enter:
   ```
   https://github.com/connormxy/homeassistant-addon-bambuddy
   ```
2. Install the [**Bambuddy** app](https://my.home-assistant.io/redirect/supervisor_addon/?addon=628c28a8_bambuddy) from the [App store](https://my.home-assistant.io/redirect/supervisor_store/).
3. (Optional) Install [one](https://my.home-assistant.io/redirect/supervisor_addon/?addon=628c28a8_bambuddy-orca-slicer-api) or [both](https://my.home-assistant.io/redirect/supervisor_addon/?addon=628c28a8_bambuddy-bambu-studio-api) **Slicer API** apps for auto-slicing and [**Obico**](https://my.home-assistant.io/redirect/supervisor_addon/?addon=628c28a8_bambuddy-obico-ml) for failure detection.
4. Start the apps — MQTT and HA API are configured automatically.
